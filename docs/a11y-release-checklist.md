# 无障碍发版回归清单（VoiceOver 实测 + App Store 无障碍标签申报依据）

> 每次提交 App Store 前在**真机**上过一遍本清单（模拟器的 VoiceOver 行为与真机有差异）。
> 全部通过后，才能在 App Store Connect 的 **Accessibility Nutrition Labels** 里如实勾选对应项——
> 虚假申报按审核指南 2.3（不实元数据）处理，宁可少勾不可虚勾。
> 打开方式：设置 → 辅助功能 → VoiceOver（建议配合三击侧边键快捷开关）。

## A. 全流程任务实测（VoiceOver 开启）

每项要求：所有交互元素可聚焦、有可理解的中文/英文朗读名、操作结果被朗读、无焦点死角。

- [ ] **冷启动 → 注册**：角色选择、隐私/条款同意（可逐字阅读全文）、设 userid、绑邮箱全程可完成
- [ ] **登录**：密码/邮箱验证码/Apple/passkey 四种方式入口可辨识可用；错误提示被朗读（绝无原始错误码）
- [ ] **盲人首页 Hub**：每张卡片朗读名与用途清晰；进入/退出各功能焦点不丢失
- [ ] **避障（导盲）**：进入后有语音确认；障碍播报与提示音正常；退出手势/按钮可达
- [ ] **看一看（识别）**：各模式（读文字/整页/纸币/颜色/找物/周围的人/光线/扫码）可切换、结果被朗读；**找椅子/沙发时报"空着/可能有人"**
- [ ] **导航**：目的地输入（含听写）、路线预览逐条朗读、开始导航后转向指令清晰
- [ ] **远程协助**：发起求助/呼叫亲友、等待态播报、接通、开关画面（隐私门控播报）、**通话文字（RTT）收发与朗读**、挂断二次确认、魔法轻点（双指双击）挂断
- [ ] **来电**：铃声+播报来电人；接听/拒绝可达；CallKit 系统界面接听后回 App 正常
- [ ] **聊天**：会话列表、消息朗读（文本/语音/图片/视频/位置）、输入与发送、群聊
- [ ] **紧急**：SOS 长按触发、30 秒倒计时播报、取消可达；摔倒检测弹层可取消
- [ ] **设置**：每个开关有状态朗读；身份切换、语言切换、应用锁、2FA、会话管理可完成
- [ ] **删除账号**：全流程可完成且确认文案清晰

## B. 系统适配项

- [ ] **动态字体**：设置 → 辅助功能 → 更大字号 调到最大，主要界面不截断、不重叠
- [ ] **深色模式**：切换后对比度仍达标（重点看自定义色）
- [ ] **减弱动态**：开启后无令人不适的动画残留
- [ ] **语音控制（Voice Control）**：主要按钮可用语音点按（名称与可见文本一致）
- [ ] **盲文点显器**（如有设备）：通话文字、聊天消息可逐条读出

## C. 双通道播报（本项目特有不变量）

- [ ] **关闭 VoiceOver**、以盲人身份操作：求助/呼叫/紧急/识别的关键状态仍被 App 自带 TTS 念出（A11y.announce 在无 VO 时会被系统丢弃——双通道是硬要求）
- [ ] 避障播报与其他语音不重叠（避障永远优先，说完补播）
- [ ] 连续收到多条通话文字时逐条顺读、不互相掐断

## D. App Store Connect 申报映射

实测全过后，Accessibility Nutrition Labels 可如实勾选：
- Supports VoiceOver ✓（A 节全过）
- Supports Voice Control ✓（B-4 过）
- Supports Larger Text ✓（B-1 过）
- Supports Dark Interface ✓（B-2 过）
- Supports Reduced Motion ✓（B-3 过）
- Supports Captions：勾选依据=通话内实时文字（RTT）与识别结果的文字呈现（如对含义有疑义，宁可不勾）
