# 真机实拍截图投放处 / Drop real device screenshots here

把你用真机（带摄像头与 LiDAR）拍的 App 截图存到**这个文件夹**，文件名按下表命名。
存好后告诉我，我会自动压成 760px 宽 JPEG、放进 `site/public/assets/shots/`、
替换官网 hero 与对应特性行的占位插画，并重新部署。

> 为什么需要文件：你在聊天里贴的图我能看见，但无法把聊天附件存成磁盘文件——
> 官网嵌入需要真实的图片文件。

## 需要的文件（优先级从高到低）

| 文件名            | 对应你发来的画面                                   | 用在官网哪里                         | 优先级 |
|-------------------|----------------------------------------------------|--------------------------------------|--------|
| `home.jpg`        | 主屏：「正前方通畅」状态条 + 求助 + 看一看/步行导航… | hero 主视觉 + 「端侧实时避障」特性行 | ★★★ 必需 |
| `look.jpg`        | 「看一看」九宫格（朗读文字/识别颜色/找东西…）        | 「场景识别」特性配图                 | ★★★ 必需 |
| `navigation.jpg`  | 「步行导航」页                                      | 可选：新增导航特性配图               | ★★ 可选 |
| `messages.jpg`    | 「消息」会话列表                                    | 「聊天·群聊·视频消息」配图           | ★ 可选 |
| `group.jpg`       | 「新建群聊」选成员页                                | 「聊天·群聊·视频消息」配图（二选一） | ★ 可选 |

原图直接放进来即可（无需自己压缩）；竖屏整屏截图最好（含状态栏也行，会自动处理）。

## 放好之后

跟我说一声，或在仓库根直接运行：

```bash
bash incoming-screenshots/apply.sh
```

脚本会：① 把每张图压成 760px 宽 JPEG → `site/public/assets/shots/`；
② 解开 `看一看 / 步行导航 / 聊天` 三行特性的注释（仅对已有图片的行）；
③ 提交并重新部署官网。幂等，可反复运行；只换你放进来的那几张。

> 在你提供文件之前，这三行特性在 `index.html` 里是**注释状态、不上线**，
> hero 与「避障」行用现有插画占位——线上不会出现任何占位/mock 内容。
