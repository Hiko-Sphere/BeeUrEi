# BeeUrEi 角色与界面设计

> 每个角色看到**不同的界面与功能**。启动后先登录、再确认角色，才进入对应主界面。
> 隶属组织 Hiko Sphere 彦穹科技 · 软件制作人 Li Yanpei Hiko。

## 进入流程

```
启动 → 安全须知知情同意 → 登录 / 注册 → 确认角色 → 进入该角色主界面
```

- **注册**时选身份：求助者（视障）/ 协助者 / 亲友；`admin` / `developer` 由后台分配（不可自助注册）。
- **登录后**显示账号角色，点「以 X 身份进入」确认后进入对应界面（不再一上来就进盲人界面）。
- **开发者**在确认页可选择以**任一角色界面**进入（便于测试）。
- 每个角色界面都可：查看账号、退出登录、切换角色（回到确认页）。

## 角色 × 界面 × 功能

### 1. 求助者（视障 blind）— 无障碍优先、大控件、VoiceOver
- **实时避障**：相机/LiDAR + 语音/震动（主界面）。
- **一键求助**：呼叫帮手 / 亲友 / 紧急。
- **步行导航**：海外 MapKit / 国内高德。
- **设置**：语速、简短播报、避障/导航开关、无障碍。

### 2. 协助者（helper / 志愿者）— 普通视觉 UI
- **在线/离线开关**（待命接听求助）。
- **来电/请求** → 接听进入**通话**：看求助者画面（对方开启时）+ 双向语音，自己**不开摄像头**。
- **协助记录 / 帮助人数**（统计）。
- **资料**：语言、时区、可服务时段。
- 通话中可**举报 / 结束**。

### 3. 亲友（family）— 面向自己的视障亲人
- **我的视障亲人**列表（绑定 / 解绑）。
- **紧急呼叫优先接听**（待命）。
- **通话**（同协助者：看画面 + 对讲）。
- （未来）亲人状态 / 位置（若对方共享）。

### 4. 管理员（admin）— 后台管理
- **用户管理**：列表、封禁 / 解封。
- **举报队列**：查看与处理。
- **录制策略**：默认关 / 保留期配置。
- **概览统计**：用户数、在线、举报数。

### 5. 开发者（developer）— 开发测试
- **角色切换器**：以任一角色界面进入。
- **后端连接**：自定义 API URL（仅此模式可改）。
- **后端统计 / 健康检查**（/api/dev、/health）。
- **避障开发者叠层**：温度 / 帧率 / ROI / 跟踪 / 分辨率。

## 实现映射

| 角色 | 主界面 View |
|---|---|
| blind | `HomeView`（现有避障 + 求助 + 设置） |
| helper | `HelperHomeView` |
| family | `FamilyHomeView` |
| admin | `AdminHomeView` |
| developer | `DeveloperHomeView`（含角色切换） |

路由：`AppRootView`（同意门 → `AuthGateView` 登录 → 恢复账号 → `RoleEntryView` 确认 → `RoleHomeView` 分发）。
